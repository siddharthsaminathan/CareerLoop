## Consolidated Bug List (Priority Order)

| # | Severity | File(s) | Bug | Root Cause | Fix |
|---|----------|---------|-----|------------|-----|
| **P0** | 🔴 INFINITE LOOP | `auth.tsx`, `ChatPage.tsx`, `ChatContext.tsx` | After onboarding completes, ProtectedRoute redirects back to /chat because AuthContext user is stale | `loadUserProfile()` only runs on app init. Onboarding doesn't refresh it. | Add `refreshUser()` to AuthContext. Call it before navigating to /brief after PROFILE_READY. |
| **P1** | 🔴 BLOCKING | `tool_registry.py`, `ChatPage.tsx` | Typing "/scan" in chat blocks for 60s | Supervisor graph runs scan synchronously | Intercept "/scan" in ChatPage → run async flow (POST /v1/scans + SSE) |
| **P2** | 🔴 WRONG DESTINATION | `auth.tsx:129` | NEW_USER redirected to /chat instead of /onboarding | ProtectedRoute targets /chat | Change redirect to /onboarding (has proper auto-start) |
| **P3** | 🔴 WRONG DEFAULT | `ChatContext.tsx:19-23` | Chat shows "I've analyzed your profile" for NEW_USER | Hardcoded PROFILE_READY message | Show neutral message when no state known yet |
| **P4** | 🟡 RACE CONDITION | `BriefPage.tsx` | Scan DONE → loadBrief() → 404 because brief not yet written | No retry logic on brief fetch | Retry 3x with 1s delay after DONE |
| **P5** | 🟡 EMPTY RESULTS | `BriefPage.tsx` | Scan returns 0 items → shows "Inbox Zero" with no action | No empty-results fallback | Check items.length === 0 → show "No matching jobs" with re-scan |
| **P6** | 🟡 NO TIMEOUT | `BriefPage.tsx` | Loading spinner hangs forever if API stalls | No loading timeout | Add 15s timeout on loading state |
| **P7** | 🟡 MISLEADING SCAN IN CHAT | `ChatPage.tsx` | Chat shows default greeting, then "/prefill=/scan" runs | ChatContext default + prefill handling | Ensure "/scan" in chat runs async scan with progress in chat |

## Verified Working (No Issues)

- ✅ `POST /v1/auth/me` — Provisions user row correctly
- ✅ `POST /v1/chat/message` — Routes to OnboardingFlow for NEW_USER, supervisor for PROFILE_READY
- ✅ `GET /v1/briefs/latest` — Returns correct BriefResponse shape
- ✅ `POST /v1/scans` — Returns run_id immediately (<200ms)
- ✅ `GET /v1/scans/{run_id}/events` — SSE stream with all event types
- ✅ `GET /v1/jobs/{job_id}` — Returns full JobDetail
- ✅ `POST /v1/jobs/{id}/save` + `POST /v1/jobs/{id}/skip` — Writes to user_job_relationships
- ✅ `POST /v1/briefs/{brief_id}/items/{item_index}/select` — Sets active context
- ✅ Onboarding flow (all 7 steps) — Properly handles LinkedIn-first and CV-first
- ✅ JWT persistence — `onAuthStateChange` + `getSession()` on every API call
- ✅ Logout — `supabase.auth.signOut()` + redirect to /login
- ✅ Profile shows real user data from API (not "Demo User")

## Summary

**7 bugs found (3 critical, 4 important).**
