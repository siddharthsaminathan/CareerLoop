# CareerLoop — Decision Compression Vision Spec (Phase 1.5)
## Owner: Hayagreev Sivakumar (CEO)

> **Status:** Phase 1.5 — CEO is building this. Depends only on discovery/search/intelligence.
> NOT dependent on Gmail, Chrome extension, interview memory, or career graph.

---

## Objective

CareerLoop should never behave like LinkedIn, a job board, an AI search engine, a spreadsheet, or an ATS dashboard.

The product objective is: **"Compress a chaotic job market into a small number of clear daily decisions."**

The user should NEVER feel overwhelmed, flooded, cognitively overloaded, or forced to compare 57 jobs manually.

The user should feel guided, strategically assisted, emotionally calm, operationally clear.

---

## Core Principle

The user is not buying job search, AI resumes, or automation.

The user is buying: **clarity, momentum, reduced uncertainty, career acceleration.**

---

## Phase 1.5 Goal

Transform **57 discovered jobs** into **5 meaningful actions.**

---

## Decision Compression Architecture

### Layer 1 — Discovery Output (DONE)
- 50–100 verified India-fit jobs
- Scored by India Fit Engine
- Deduped, company-verified
- **Problem:** Raw intelligence overload

### Layer 2 — Search Modes
Every user operates in one runtime mode. These alter filtering strictness, application volume, scoring thresholds, recommendation behavior, and follow-up aggressiveness.

| Mode | Behavior | Filtering | Volume |
|------|----------|-----------|--------|
| **EXPLORE** | Low-pressure discovery. Weekly digest. Minimal interruptions. | High | Low |
| **UPGRADE** | Working professionals. Only switch-worthy roles. | Strict | Low |
| **HUNT** | High-volume search. Aggressive application throughput. | Moderate | High |
| **EMERGENCY** | Urgent employment. Maximum velocity. Reduced strictness. Strong follow-up. | Low | Maximum |

### Layer 3 — Aggression Slider
Independent from modes. Affects surfaced job count, score thresholds, recommendation confidence, stretch opportunity willingness.

- Aggression 1: safest only, highest fit only
- Aggression 5: broader expansion, more speculative, higher throughput

### Layer 4 — Role Track Clustering
DO NOT show 57 isolated jobs. Cluster into strategic tracks.

Example:
- **Track A:** AI Product Engineer
- **Track B:** LLM Platform Engineer
- **Track C:** Growth/Product Analytics

Each track gets one positioning direction, one application pack, one strategy narrative.

This dramatically reduces resume generation count, token cost, user fatigue, and repeated decisions.

### Layer 5 — Daily Triage Board
The primary UX surface. NOT a feed. NOT a dashboard. A morning operational briefing.

**Output Format:**
- **5 worth applying**
- **2 high upside**
- **3 backup options**
- **41 rejected automatically**

Rejection reasons: low salary, startup risk, notice-period mismatch, weak company signals, poor fit.

The user should feel: "CareerLoop already thought through the market for me."

### Layer 6 — Job Memory Cards
Every surfaced role becomes a persistent memory object storing: why this role matters, risks, recruiter likelihood, salary confidence, company intelligence, application state, follow-up state, attached application pack.

The user should be able to ask: "Why is this a good fit? What are the risks? How should I position? Is this company worth my time? Why did this outrank another job?"

The system should answer conversationally.

### Layer 7 — Decision UX Rules
- **RULE 1:** Never show 50 jobs at once.
- **RULE 2:** Never require spreadsheet thinking.
- **RULE 3:** Never force the user to compare manually.
- **RULE 4:** Never expose raw scoring systems directly.

The UX should feel human and guided.

### Layer 8 — Daily Ritual
The product habit: Morning → CareerLoop Brief. Midday → Applications/follow-ups. Night → Interview prep/reflection.

The product should become a career operating rhythm.

### Layer 9 — Success Metrics
The goal is NOT jobs shown. The goal is **decisions completed.**

Track: applications approved, applications submitted, follow-ups sent, jobs rejected intentionally, time-to-decision, decision fatigue reduction.

### Layer 10 — Anti-Goals
DO NOT create LinkedIn feed behavior, infinite scrolling, noisy AI reasoning exposure, spreadsheet UI, or analytics-heavy enterprise UX.

This is consumer operational UX.
