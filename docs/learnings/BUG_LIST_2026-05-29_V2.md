## P0 — BLOCKS USER FROM USING THE PRODUCT

| # | Bug | File | Line | What Happens | Fix |
|---|-----|------|------|-------------|-----|
| **P0-01** | **Application packs never generated** | `session/tool_registry.py` | 883-921 | `prepare_application_pack()` creates a QUEUED row in background_runs but NEVER calls `PackageAssembler.assemble_package()`. User is told "pack queued" but gets NO resume PDF, NO cover note, NO recruiter DM, NO screening answers. **application_packs table has 0 rows.** | Wrap pack generation in daemon thread (like scan), call PackageAssembler.assemble_package() |
| **P0-02** | **Resume editing produces no output** | `session/tool_registry.py` | 923-953 | `edit_application_pack()` creates a QUEUED row but NEVER calls ResumeCouncil. User says "edit my resume" and gets a pack ID with nothing behind it. | Call ResumeCouncil in the edit path |
| **P0-03** | **JWT expiry = loading forever** | Frontend `BriefPage.tsx` + `api.ts` | api.ts:32-64 | When JWT token expires, API returns 401. Frontend `loadBrief()` retries 3× but NEVER refreshes the token. Spinner spins until all 3 attempts × 15s timeout = 45s, then shows empty state. | Add token refresh or auto-redirect to login on 401 |
| **P0-04** | **First onboarding message discarded** | `onboarding/onboarding_flow.py` | 82-85 | `_handle_idle()` ignores text content — ALWAYS returns greeting and asks for name. If user already types their name in first message, they must repeat it. | Check if text looks like a name, route to _handle_identifying directly |
