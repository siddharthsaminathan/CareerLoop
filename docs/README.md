# CareerLoop Documentation Portal

Welcome to the CareerLoop documentation portal. To maintain a clean and highly structured codebase, all product documents, engineering designs, sprints/backlogs, and retrospective learnings are organized into exactly four specialized directories.

## Directory Structure

### 📂 [Product](file:///Users/siddharthsaminathan/Projects/CareerLoop/docs/product/)
> High-level strategy, target personas (ICP), feature lists, and canonical vision documents.
* **Core File:** [PRD.md](file:///Users/siddharthsaminathan/Projects/CareerLoop/docs/product/PRD.md) — The single source of truth for the product requirements.
* **Read when:** Aligning on business objectives, geographical constraints, or emotional states of the target user.

### 📂 [Engineering](file:///Users/siddharthsaminathan/Projects/CareerLoop/docs/engineering/)
> Locked system architecture specifications, micro-system designs, model routing configurations, and schemas.
* **Core File:** [CANONICAL_ARCHITECTURE.md](file:///Users/siddharthsaminathan/Projects/CareerLoop/docs/engineering/CANONICAL_ARCHITECTURE.md) — The locked 8-system Resume Council architecture.
* **Read when:** Modifying LangGraph orchestrations, adding LLM schemas, or updating template AST parsers.

### 📂 [Tech Backlog](file:///Users/siddharthsaminathan/Projects/CareerLoop/docs/tech-backlog/)
> Sprint logs, active engineering trackers, checklist metrics, and implementation planning artifacts.
* **Core File:** [TRACKER.md](file:///Users/siddharthsaminathan/Projects/CareerLoop/docs/tech-backlog/TRACKER.md) — Current sprint focus, system status tables, and blockers.
* **Read when:** Updating system completion percentages, checking Sprint B1-B8 blockers, or starting new session tasks.

### 📂 [Learnings](file:///Users/siddharthsaminathan/Projects/CareerLoop/docs/learnings/)
> Detailed code quality audits, daily dev blogs, retrospective fuckups, delta forensics, and root-cause analyses.
* **Core File:** [FUCKUPS.md](file:///Users/siddharthsaminathan/Projects/CareerLoop/docs/learnings/FUCKUPS.md) — Deep-dive retrospectives into past engineering mistakes to avoid regressions.
* **Read when:** Conducting code reviews, understanding previous design failures, or analyzing compiler tailoring deltas.
