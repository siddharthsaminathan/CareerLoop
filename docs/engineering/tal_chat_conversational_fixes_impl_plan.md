# Engineering Plan: Making CareerLoop Truly Conversational

This document outlines the concrete technical changes required to eliminate hardcoded robotic interceptions, expand the semantic routing dictionary with the `UPDATE_PROFILE` intent, and implement conversational state-machine loops.

---

## 1. Concrete Changes

### A. Upgrade Intent Classification (`careerloop/llm_chat.py`)
Add a new intent, `UPDATE_PROFILE`, to the `ChatIntentAgent` system prompt:
```
- UPDATE_PROFILE: User wants to search for, sync, import, or update their profile details from LinkedIn or a resume (e.g. 'go find me on linkedin', 'update my profile from linkedin', 'sync linkedin', 'import profile', 'search for my profile').
```
Add extraction of `target_name` if present (e.g., *"go find me on linkedin"*).

---

### B. Dynamic Onboarding Entry Points (`careerloop/session/supervisor_graph.py`)
We will remove the rigid "brief ready" interceptor and allow the `UPDATE_PROFILE` intent to transition users from active use states directly back into the onboarding search engine.

#### 1. Update Active States (`_handle_active_state`)
```python
    if intent == "UPDATE_PROFILE":
        reply = "Sure! I can search for your profile and sync your LinkedIn details right now. What is your full name?"
        return {
            "current_state": UserState.ONBOARDING_IDENTIFYING,
            "assistant_response": reply,
            "messages": [AIMessage(content=reply)],
            "temp_profile_data": profile_data,
        }
```

#### 2. Remove Rigid Override on `SCAN_JOBS`
Instead of hard-overwriting the LLM reply with `"Today's brief is already ready!"` immediately when `intent == "SCAN_JOBS"`, we will only trigger the reminder if the user explicitly asks to crawl again, using a much more helpful and conversational reply generated by the LLM.

#### 3. Update the Intent Router State Handlers
Add the `UPDATE_PROFILE` intent handler across all post-onboarding states (`REVIEWING_JOB`, `PACK_READY`, `AWAITING_APPLICATION_CONFIRMATION`, `APPLIED`):
```python
        elif result["intent"] == "UPDATE_PROFILE":
            reply = "Let's update your profile! What is your full name so I can search for your LinkedIn profile?"
            return {
                "current_state": UserState.ONBOARDING_IDENTIFYING,
                "assistant_response": reply,
                "messages": [AIMessage(content=reply)],
                "temp_profile_data": result.get("profile_data", {}),
            }
```

## 2. Verification Steps

1.  **Conversational Sync Test:** Start the CLI in a completed state (`BRIEF_AVAILABLE`), type *"go find me on linkedin"*, and verify that the system dynamically shifts its state to `ONBOARDING_IDENTIFYING` and prompts: *"Sure! I can search for your profile... What is your full name?"*.
2.  **No More Hardcoded Overwrite Test:** Verify that saying *"run a search"* returns a highly contextual conversational answer instead of a hardcoded robot sentence block.
